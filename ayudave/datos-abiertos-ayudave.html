Datos abiertos AyudaVE

AyudaVE publica endpoints publicos para recibir y sincronizar reportes comunitarios, refugios, centros de acopio, puntos de ayuda y datos reducidos de personas.

Los feeds excluyen contactos privados, secretos, cedulas y datos sensibles. Toda informacion externa debe mostrarse con atribucion y revalidarse antes de mover recursos.

Si integras sistemas

Consume el feed JSON, guarda la fecha de ultima actualizacion y usa sincronizacion incremental.

Ver endpoints explicados
  1. Consume api.php?action=export_public para obtener el snapshot publico completo.
  2. Guarda el campo generatedAt o el mayor updatedAt recibido.
  3. En la siguiente llamada usa since o updated_since para bajar solo cambios.
  4. Consulta sync_status para saber si una fuente esta fresca, desactualizada o con error.
  5. Si replicas datos, muestra "AyudaVE" y la fuente original. No uses registros sin validar para movilizar recursos.

Endpoints publicos

GET

/api.php?action=export_public

Snapshot JSON principal con reportes y puntos de ayuda publicables.

  • since o updated_since: fecha ISO 8601 para sincronizacion incremental.
GET

Estado tecnico legible

Estado de base de datos, ultimas sincronizaciones, fuentes y conteos. JSON: sync_status.

GET

/api.php?action=metadata

Catalogos de tipos, estados, niveles de confianza y fuentes disponibles.

GET

/api.php?action=people

Personas desaparecidas/localizadas en modo reducido y paginado.

  • limit: maximo de registros por pagina.
  • offset: desplazamiento para paginar.
  • status y q: filtros opcionales.
GET

/api.php?action=external_metrics

Metricas agregadas externas de personas. No importa fichas personales en el mapa.

GET

/api.php?action=export_csv&dataset=...

Descargas CSV para hojas de calculo o procesos simples.

Ejemplo de consumo

const url = "https://ayudave.mranalytics.info/api.php?action=export_public";
const res = await fetch(url);
const data = await res.json();

const puntosConfiables = data.helpPoints.filter((punto) =>
  ["verified_origin", "community_confirmed"].includes(punto.trustLevel)
);

const reportesActivos = data.reports.filter((reporte) =>
  reporte.status !== "Resuelto" && reporte.lat && reporte.lng
);

Estados de validacion

  • Sin validar: visible para coordinar confirmacion, no para movilizar recursos sin revisar.
  • Confirmado: validado por fuente externa o seguimiento comunitario.
  • Resuelto: necesidad atendida o ya no activa.
  • Abierto: punto operativo informado por fuente externa.
