Datos abiertos AyudaVE

AyudaVE publica un feed JSON para sincronizar reportes comunitarios, refugios, centros de acopio y puntos de ayuda en Venezuela.

Abrir feed publico

Endpoint

GET https://example.org/api.php?action=export_public

Sincronizacion incremental: api.php?action=export_public&since=2026-06-28T00:00:00Z devuelve solo registros modificados desde esa fecha ISO 8601. Tambien acepta updated_since.

Metadata para integraciones: api.php?action=metadata

Estado de sincronizacion: api.php?action=sync_status

El feed excluye contactos personales y secretos. Incluye tipo, area, ciudad, prioridad, estado, nivel de confianza, coordenadas, fuente, URL de fuente e identificador externo.

Contrato JSON Schema: ayudave-public-export.schema.json

Contrato OpenAPI: openapi.json

Descargas CSV: reportes · puntos de ayuda

Estados de validacion

  • Sin validar: usar para visibilidad inicial, no para movilizar recursos sin confirmar.
  • Confirmado: validado por fuente externa o seguimiento comunitario.
  • Resuelto: necesidad atendida o ya no activa.
  • Abierto: punto operativo informado por fuente externa.

Niveles de confianza

  • verified_origin: dato confirmado por su fuente original.
  • external_pending: dato de fuente externa que debe validarse localmente.
  • community_confirmed: dato confirmado por moderacion o seguimiento comunitario.
  • community_pending: dato comunitario recibido, pendiente de validacion.
  • resolved: dato marcado como resuelto.

Ejemplo de consumo

const res = await fetch("https://example.org/api.php?action=export_public");
const data = await res.json();

const puntosConfiables = data.helpPoints.filter((punto) =>
  ["verified_origin", "community_confirmed"].includes(punto.trustLevel)
);

const reportesActivos = data.reports.filter((reporte) =>
  reporte.status !== "Resuelto" && reporte.lat && reporte.lng
);

Si replicas datos, muestra atribucion a AyudaVE y a la fuente original. Revalida condiciones antes de mover recursos.
